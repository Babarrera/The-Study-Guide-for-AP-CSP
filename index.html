📘 AP Computer Science Principles – Complete Study Guide

Includes all original terms + every missing term identified from the exam.

Unit 1 & Big Idea 1: Creative Development

Collaboration & Development Processes

  • Computing innovation: an innovation that uses a program as a key part of their function.
    • Physical examples: Robots, Tablets, smart tech.
    • Non-physical examples: Social Media, Video Games.
  • Collaboration: allows for diverse ideas and thoughts that cater to different people.
  • Pair programming: when two people share one computer and take turns coding.
  • Iterative development process: develop working prototypes of a program and go back through the cycle to redevelop the program.
  • Incremental development process: break a problem into small parts and then reassemble the solution when each party is fixed.
  • User feedback: Input from users that helps developers refine and improve a program.
  • Prototype: An early model of a program used for testing and refining ideas before full implementation.
  • Version Control System (VCS): A tool that helps track and manage changes to code, allowing multiple developers to collaborate.
  • Event-driven programming (➕ missing): Program execution that waits for user actions (clicks, key presses) to trigger code.
  • User-centered design (➕ missing): Designing a program by focusing on the needs, wants, and limitations of end users.
  • Development environment (IDE) (➕ missing): Software that helps write, debug, and run code.

Program Function & Behavior

  • Purpose: What a program is designed to do (ex: solving problems, creative expression).
  • Program events: an action that gives a program data to respond to.
  • Program behavior: how a program will respond to a user interacting with it.
  • Program functionality: How well a program performs its intended task and meets user needs.
  • Documentation: a description of how something in your program works.
  • Comments: program doc. written directly into the program itself.

Errors & Methods of Fixing

  • Logic errors: unexpected behavior in program's output.
  • Syntax errors: the code does not work properly because it is typed or written incorrectly.
  • Run-time errors: error occurs while code runs.
  • Overflow errors: the numbers are too big for the computer.
  • Hand tracing: manually tracking your variables' values as your program goes along.
  • Print statements: printing out values to make sure they're correct.
  • Debugging: The process of identifying and fixing errors in a program.
  • Edge case: An unusual or extreme input that tests the limits of a program's behavior.

Unit 2 & Big Idea 2: Data

Binary & Data Representation

  • Bits: Computing devices store data in bits (binary digits).
  • Binary numbers: 8 bits = 1 byte.
  • Hexadecimal: used for RGB color codes & it uses Base 16.
  • ASCII code: converts text to binary format.
  • Analog data: measured continuously & changes smoothly.
  • Digital data: measured digitally and leaves out extra data by simplifying the data collected (form of abstraction).
  • Abstraction: reduces complexity by only focusing on the most important parts & hiding the irrelevant parts from the user.
  • Sampling (➕ missing): Measuring an analog signal at discrete time intervals to convert it to digital.
  • Sample rate (➕ missing): Number of samples taken per second (e.g., 44.1 kHz for audio).
  • Bit depth (➕ missing): Number of bits used per sample (affects precision).

Data Compression

  • Lossless compression: less compression & better file quality. No data is lost; original can be perfectly reconstructed.
  • Lossy compression: more compression & worse file quality. Some data is discarded to achieve smaller file sizes.
  • Compression ratio: original size : compressed size (Example: 10 MB → 2 MB = 5:1 ratio).

Extracting & Managing Information

  • Metadata: data about data. Examples include image resolution, file size, and creation date.
  • Data mining: examining very large data sets to find information.
  • Transforming data: editing or modifying data (ex: doubling every number).
  • Cleaning data: making data uniform w/o changing meaning (ex: correcting misspelled words).
  • Filtering data: Keep only rows that meet a condition, e.g., age > 18.
  • CSV: Storing tabular data (rows/columns).
  • Correlation (➕ missing): A relationship between two variables (e.g., ice cream sales and temperature).
  • Causation (➕ missing): When one variable directly causes a change in another (correlation does not imply causation – tested heavily).
  • Outlier (➕ missing): A data point that differs significantly from other observations.
  • Data set (➕ missing): A collection of related data records.
  • Pattern (➕ missing): A regular or repeated arrangement in data (discovered through data mining).

Unit 3 & Big Idea 3: Algorithms & Programming

Logic & Problem Solving

  • Algorithm: An algorithm has instructions that accomplish a task or solve a problem; created using sequencing, selection, and iteration.
  • Sequencing: all of the code is executed in the order they are written in.
  • Selection: if statements; have conditions that need to be met for the selection to run.
  • Iteration: "repeat n times" or "repeat until".
  • Heuristic: An approach to problem-solving that uses a practical method for a sufficient immediate solution.
  • Undecidable problem: A problem for which no algorithm can provide a correct answer for all cases.
  • Binary search: works by splitting the data set and eliminates 1/2 the data with each round of splitting.
  • Linear search (➕ missing): Searching a list by checking each element one by one until the target is found.
  • Nested conditional (➕ missing): An IF statement inside another IF statement.
  • Nested loop (➕ missing): A loop inside another loop.
  • Infinite loop (➕ missing): A loop that never terminates (common logic error).
  • Efficiency (➕ missing): A measure of how many steps or how much memory an algorithm uses (often "reasonable" vs. "unreasonable" time).

Programming Constructs

  • Variables: placeholders for values that a program can use.
  • Lists (Arrays): an ordered sequence of elements.
  • AP Pseudocode Index: Important: AP Pseudocode has an index that starts at 1!
  • MOD (%): gives you the remainder of 2 #'s.
  • Procedures: programming instructions that are also called methods or functions.
  • Parameters: input variables of a procedure.
  • Arguments: a procedure call with defined values.
  • API (Application Programming Interface): A tool that allows programs to communicate and access data.
  • String (➕ missing): A sequence of characters.
  • String concatenation (➕ missing): Joining two or more strings together.
  • Boolean expression (➕ missing): An expression that evaluates to either TRUE or FALSE.
  • Logical operators (➕ missing): AND, OR, NOT (see reference sheet).
  • Return value (➕ missing): The value that a procedure sends back to where it was called (different from DISPLAY).
  • Procedural abstraction (➕ missing): Giving a procedure a name so you can use it without knowing its internal details.
  • Data abstraction (➕ missing): Using a list to manage complexity (hiding how individual pieces are stored).
  • Input (➕ missing): Data provided to a program (from user, file, or sensor).
  • Output (➕ missing): Data produced by a program (displayed, saved, or sent).

Unit 4 & Big Idea 4: Computer Systems & Networks

Internet Interconnection

  • Computer network: when multiple computing devices communicate with each other.
  • Data packets: Data on the internet is split into data packets with metadata attached to tell routers information.
  • Routing: the process of finding the best path to deliver information.
  • Path: sequences of connected computing devices (routers) that begin at sender and end at receiver.
  • Bandwidth: the rate of data transfer from one device to another; megabits per second.
  • Redundancy: duplication of things; helps make the internet fault tolerant.
  • Fault tolerant: something can still function even w/ a partial malfunction.
  • Latency (➕ missing): The time delay between sending and receiving data (often tested alongside bandwidth).
  • Router (➕ missing): A device that forwards data packets between networks.
  • Scalability (➕ missing): The ability of a system to handle growing amounts of work by adding resources.
  • Packet switching (➕ missing): The method of breaking data into packets and sending them independently (core internet design).

Protocols & Addresses

  • Protocol: set of rules.
  • IP (Internet Protocol): Breaks data down into packets and addresses them with source and destination information.
  • TCP (Transmission Control Protocol): Provides reliable, ordered, and error-checked delivery of a stream of packets.
  • UDP: Used for communication where speed is prioritized over error-checking.
  • DNS (Domain Name System): Translates human-readable domain names into IP addresses.
  • HTTP: Protocol used by web browsers to fetch webpages.
  • IPv4 vs. IPv6: IPv4 is 32 bits; IPv6 is 128 bits, created to solve the address shortage.
  • Client-server model (➕ missing): One device (server) provides services to other devices (clients).
  • Peer-to-peer (P2P) (➕ missing): Devices communicate directly without a central server.
  • Net neutrality (➕ missing): The principle that ISPs should treat all data equally (social/ethical tie-in).

Parallel & Distributed Computing

  • Sequential computing: traditional programming where each program is processed at a time.
    • Sequential solution: Takes as long as the sum of all steps: a+b+c.
  • Parallel computing: when program is broken into smaller operations and processed at the same time using multiple processors.
    • Speedup: sequential solution time / parallel solution time.
  • Distributed computing: multiple devices communicate together to run a program.

Unit 5: Impact of Computing

Social & Ethical Concerns

  • Digital divide: gaps between those who have access to the internet and those who do not.
  • Computing bias: computing innovations can reflect existing racial/gender/etc. biases.
  • Crowdsourcing: getting a large amount of input or information from people on the Internet.
  • Citizen science: scientific research that the general population helps to conduct.
  • Bias in data collection (➕ missing): When the way data is gathered systematically favors certain outcomes (e.g., survey sent only to one group).

Intellectual Property

  • Copyright: the person who created something determines who uses their creation.
  • Creative Commons: copyright license for creators to give others the ability to use their work.
  • Open-sourcing: work is freely shared, distributed, and modified.
  • Fair use (➕ missing): A legal doctrine allowing limited use of copyrighted material without permission (for education, criticism, etc.).
  • License (➕ missing): A legal agreement that specifies how software or creative work can be used.

Security & Encryption

  • PII (Personally Identifiable Information): information that can be used to identify you.
  • Malware: malicious software that takes control of a system.
  • Phishing: tricks people into giving personal information away by pretending to be a trustworthy group.
  • Encryption: encoding data to prevent others from accessing it.
  • Symmetric key encryption: one key for both encrypting & decrypting.
  • Public key encryption: public key to encrypt & private key to decrypt.
  • Virus (➕ missing): Malware that spreads by inserting itself into other programs.
  • Spyware (➕ missing): Malware that secretly collects user information.
  • Ransomware (➕ missing): Malware that locks files and demands payment.
  • Keylogging (➕ missing): Recording keystrokes to steal passwords.
  • Multi-factor authentication (➕ missing): Using two or more verification methods (password + text code).
  • SSL/TLS (➕ missing): Protocols that encrypt web traffic (HTTPS uses this).
  • Cipher (➕ missing): An algorithm for encrypting and decrypting data.
  • Decryption (➕ missing): The process of converting encrypted data back to original form.

Other Important Concepts (appear on the exam)

  • Simulation: A program that models a real-world system using random numbers and rules.
  • Expected value: The average outcome of a random process over many trials.
  • Reasonableness: Whether a problem can be solved in a practical amount of time (polynomial vs. exponential).
  • Turing completeness: The idea that a programming language can simulate any algorithm.
  • Empirical data: Data collected through observation or experiment.

Arithmetic & Relational Operators

Text SyntaxExplanation

DISPLAY(expression)Displays the value followed by a space.
a MOD bEvaluates to the remainder when a is divided by b.
RANDOM(a, b)Returns a random integer from a to b, inclusive.
NOT conditionTrue if condition is false.
a AND bTrue if both are true.
a OR bTrue if either or both are true.

List Operations

Note: If a list index is less than 1 or greater than the length of the list, the program terminates.

  • aList[i]: Accesses the element at index i.
  • INSERT(aList, i, value): Shifts values at i and greater to the right; length increases by 1.
  • APPEND(aList, value): Value is placed at the end of the list.
  • REMOVE(aList, i): Removes item at i and shifts values to the left.
  • LENGTH(aList): Evaluates to the number of elements.
  • FOR EACH item IN aList: Executes a block once for each assignment of item.

Procedures & Robot Commands

  • PROCEDURE procName (parameter1, ...): Defines a procedure with zero or more arguments.
  • RETURN (expression): Ends the procedure and returns the value of expression.
  • MOVE_FORWARD(): Moves robot one square forward.
  • ROTATE_LEFT() / ROTATE_RIGHT(): Rotates robot 90 degrees.
  • CAN_MOVE(direction): True if there is an open square in the relative direction.
